Open Source Cloud Computing

What is OpenStack

  • "Operating system for data centers"
  • Manages pools of resources (compute, storage, networking, ...)
  • Makes those resources available to "tenants"
  • Get in touch
    • #openstack on freenode IRC
    • www.openstack.org

OpenStack "Mitaka" Release Highlights

  • Scalability
    • e.g. Identity fernet tokens, Compute Cells v2 improvements
  • Manageability
    • e.g. simplified Compute configuration and Identity bootstrapping
  • User Experience
    • e.g. continued CLI unification, simplified virtual network creation

OpenStack "Newton" Release Highlights

  • Scalability
    • e.g. easier scale up/down for Compute, Dashboard, and Object Storage
  • Resilience
    • e.g. convergence in Orchestration by default, HA improvements
  • Versatility
    • e.g. container-related enhancements (orchestration tools, networking)

"OpenStack on OpenStack"

What is TripleO

  • Deployment and management of OpenStack
  • Using OpenStack's own tools/mechanisms/facilities
  • Operator tool set intersecting with tenant tool set
    • Both worlds benefit from enhancements
  • Get in touch
    • #tripleo on freenode IRC
    • tripleo.org

TripleO - state at DevConf '16

  • 5 predefined node roles to build your cloud with
    • Incl. seamless deployment of Ceph storage
  • Network traffic isolation
    • All the way to physical NICs level
  • Updates between minor versions

TripleO - done since DevConf '16 (highlights)

  • IPv6 support
  • TLS for API endpoints
  • Upgrades between major releases
  • Full deployment composability
    • Selection of services to deploy
    • Definition of custom node roles

TripleO - what's in the works (highlights)

  • Undercloud deployment via the same mechanism as overcloud
  • Containerized deployment
  • Flexible major upgrades framework
  • Additional TLS-enabled scenarios
    • Incl. certificate management via FreeIPA
